# Coding Practice Principles

MXZTAR-forge follows practical problem-solving discipline.

## Core rules

1. Code is for humans first, machines second.
2. Every name in code should earn its keep.
3. Bugs are usually broken assumptions.
4. Good functions have one clear responsibility.
5. Separate what the code does from how it does it.
6. Duplication is a drift risk.
7. Abstractions should remove difficulty, not hide reality.
8. Error handling is part of design.
9. Data shape is destiny.
10. Programming is controlled change.

## UI rules

- No dead UI.
- No frozen UI.
- No long work on the Qt main thread.
- One working control is better than ten decorative controls.
- Every visible button needs a handler, workflow path, output path, feedback, error state, and verification method.
